## 适合谁

- 需要管理大量本地视频、剧集和分段内容的用户
- 需要把本地磁盘、挂载盘、OpenList 或远程来源统一纳入管理的人
- 需要搜索、收藏、追看、随机抽取内容的重度观影用户
- 需要下载、转码、扫描、补全元数据的媒体整理用户

## 核心能力

### 1. 媒体库管理
- 按分类组织视频和剧集
- 支持批量扫描、文件监控、自动入库
- 支持读取本地 NFO、补齐媒体信息
- 支持海报、缩略图、片名、时长、分辨率、编码等信息展示

### 2. 搜索与筛选
- 全文搜索标题、描述、文件名等信息
- 高级筛选支持分类、标签、类型、画质、时长、评分等条件
- 支持搜索建议、保存搜索与结果高亮

### 3. 播放与历史
- 快速查看最近播放与详细历史记录
- 支持从列表、收藏、历史、搜索结果中直接播放

### 4. 收藏与标签
- 支持视频和剧集收藏
- 支持标签管理、批量标签操作、标签筛选
- 便于把内容按兴趣、题材或自定义规则二次整理

### 5. 下载与转码
- 内置通用下载管理
- 支持 BT / 磁力相关任务管理
- 支持网页视频下载
- 支持视频格式转换与并发任务管理

### 6. 远程来源与索引
- 支持来源索引与局部同步
- 适配 OpenList 等挂载来源
- 可查看索引文件、目录、任务历史和引用关系
- 适合统一管理本地与远程媒体来源

### 7. 随机观影
- 提供随机抽取内容的“盲盒”入口
- 可直接播放单个视频，或进入剧集详情继续选择
- 适合快速开看，不用先翻库



## 产品亮点

- 桌面端本地应用，数据可控
- 覆盖从入库、整理、搜索到播放的完整链路
- 同时兼顾普通本地库和挂载来源场景
- 适合长期积累和维护个人媒体资产

## 目录命名建议
- 电影建议使用 电影名 (年份) 或 电影名.年份。
- 剧集建议使用 剧名/Season 1/S01E01.mkv 或 剧名/S01E01.mkv。
- 如果已有 NFO，放在视频同目录或剧集根目录，后台补齐会优先尝试读取。
- 如果知道 TMDB ID，可以在目录名或文件名中写入 {tmdb-12345}、{tmdbid=12345}、{tmdbid=12345} 或 {tmdbid-12345}，提高匹配准确性。

| 内容类型   | 推荐格式                                             | 说明                                                         |
| ---------- | ---------------------------------------------------- | :----------------------------------------------------------- |
| 单个视频   | 电影名 (年份) {tmdbid-电影ID}/电影名.年份.清晰度.mkv | 有独立文件夹时，把 TMDB ID 放在文件夹名最稳；没有文件夹时，放在视频文件名也可以。 |
| 剧集根目录 | 剧名 (首播年) {tmdbid=剧集ID}                        | 这里应填写 TMDB 的剧集 ID，不是某一季的 ID。多季剧集可以从根目录继承这个 ID。 |
| 季目录     | `剧名 S01`、`剧名 S02` 或 `Season 1`                 | 系统支持 `第一季`、`第1季` 等写法，但 `S01`、`S02` 最清晰。  |
| 分集文件   | `剧名.S01E01.标题.mkv` 或 `剧名 - S01E01 - 标题.mkv` | 多季内容必须在文件名或季目录中体现季号，否则容易被当成同一季或普通视频。 |
| 单季剧集   | 剧名 (年份) {tmdbid=剧集ID}/剧名.S01E01.mkv          | 只有一季时可以不单独建 `S01` 文件夹；如果文件很多，建 `S01` 文件夹更容易维护。 |

#### 单个视频示例：

```
铁血战士 (1987) {tmdb-106}/
  铁血战士.1987.2160p.REMUX.mkv

超级战舰 (2012) {tmdbid=44833}.mkv
```

#### 多季剧集示例：

```
食贫道充电纪录片 (2023) {tmdbid=284720}/
  食贫道充电纪录片 S01/
    食贫道充电纪录片.S01E01.迷失东京.mp4
  食贫道充电纪录片 S02/
    食贫道充电纪录片.S02E01.迦南孤儿.mp4
```

## 创建分类：

| 方式       | 适合场景                         | 特点                                                         |
| ---------- | -------------------------------- | ------------------------------------------------------------ |
| 本地目录   | 文件就在本机硬盘且数据量不是很大 | 扫描目录并直接入库，可配合文件监控自动发现新增文件。         |
| OpenList   | 已有 OpenList 服务和搜索索引     | 优先读取 OpenList 索引，再映射到本地挂载路径播放（也可采用下方的方式）。 |
| 数据源索引 | 普通挂载盘                       | 先在应用里扫描挂载目录生成公共索引，再用索引创建分类。       |

### 本地目录方式

1. 进入分类页面，点击创建分类。
2. 选择本地目录方式，填写分类名称并选择根目录。
3. 确认后系统会扫描目录，识别视频文件、电影和剧集。
4. 扫描完成后进入分类查看内容。

### OpenList 方式

OpenList 方式适合已经通过 OpenList 管理远程文件，并且 OpenList 搜索索引可用的场景。

1. 先在 **设置 - OpenList** 中启用 OpenList。
2. 填写服务地址、Token 和默认本地挂载路径。
3. 使用“测试连接”和“测试路径”确认服务可访问。
4. 创建分类时选择 OpenList，填写 OpenList 路径和本地挂载路径。
5. 创建后在后台看板或分类操作中执行索引同步。

注：这种方式也可采用第三种方式进行，第三种方式更具有普遍适用性，但要求在本机有挂载盘

### 数据源索引方式

1. 创建数据源：打开 **数据源索引** 页面，填写数据源名称，选择平台，例如 CloudDrive2，并选择挂载根目录。
2. 生成索引：点击 **生成索引**。系统会扫描目录，记录目录、文件、视频数量和失败路径。
3. 用索引创建分类：创建分类时选择数据源索引方式，选择数据源和索引路径。路径可以指向整个数据源，也可以指向其中某个子目录。
4. 同步分类索引：进入数据源索引界面选择局部更新功能，选择要更新的路径（最好路径要大才能对比出差异，如果是小数据量更新也可以指定具体目录此时不太好对比差异，如果是新增还好）。
